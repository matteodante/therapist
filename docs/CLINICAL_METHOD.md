# Clinical method

## Status

The method is an evidence-informed product design, not a validated clinical
treatment. Every skill is marked as requiring professional review.

## Backbone

The initial backbone is **transdiagnostic CBT guided by collaborative
formulation**, complemented by selected principles from:

- foundational helping skills;
- behavioural activation;
- motivational interviewing;
- ACT;
- structured review and outcome monitoring.

## Process

```text
engage
→ understand the current concern
→ assess progressively
→ formulate tentatively
→ agree a focus and goal
→ select an intervention
→ review observable outcome
→ revise
```

## Formulation

A useful formulation may include:

- current concern;
- trigger;
- meaning;
- thought;
- emotion and body response;
- protective behavior;
- short-term benefit;
- long-term cost;
- context;
- strengths and exceptions;
- alternative explanations.

It must remain collaborative, culturally aware, revisable, and proportionate.

## Competence rather than scripts

The project uses Markdown protocols, but fidelity is not defined as reciting
phrases. Evaluation should assess observable competencies:

- empathy and collaboration;
- guided discovery;
- epistemic humility;
- pacing;
- appropriate formulation;
- intervention rationale;
- outcome review;
- rupture repair;
- scope awareness.

## Protocol authoring rule

Each skill must state:

- when to use;
- when not to use;
- assessment needed;
- procedure;
- pitfalls;
- verification;
- sources;
- review status.

## Human review

Before clinical or commercial claims, protocol packs require:

- review by qualified professionals;
- review by people with lived experience;
- multilingual review;
- longitudinal testing;
- adverse-effect monitoring;
- public versioning and change logs.
