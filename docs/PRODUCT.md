# Product definition

Therapist is a persistent psychological support agent that aims to reproduce the
**process discipline** of good psychological work without claiming to be a
licensed clinician.

## Product promise

Therapist learns a person's history with explicit uncertainty and uses that
understanding to:

- listen more accurately;
- ask better questions;
- identify recurring emotional, cognitive, behavioral, and relational cycles;
- formulate hypotheses collaboratively;
- select proportionate exercises;
- check what actually helped;
- revise memory and formulation when corrected.

## Desired user experience

The user should experience continuity rather than novelty:

> "It does not merely answer my latest message. It remembers the path, notices
> patterns, asks one relevant question, and changes its mind when I correct it."

## Product principles

- understanding before intervention;
- one useful question at a time;
- facts separated from hypotheses;
- alternatives and counterevidence;
- user correction over model inference;
- interventions connected to maintaining mechanisms;
- outcomes reviewed rather than assumed;
- autonomy over engagement;
- transparency and data portability;
- no unsupported clinical claims.

## Initial distribution

The first edition is open source and self-hosted. A hosted SaaS edition may
follow later, using the same protocol and data formats.

## Success measures

Therapist is not optimized for daily active use or time spent. Success means:

- fewer invented memories;
- more relevant recall;
- formulations recognized by users;
- appropriate correction after misunderstanding;
- techniques selected for a reason;
- reported behavior or functioning changes;
- low emotional dependency;
- clear escalation beyond the product's competence.
