# Foundational helping

## When to use

Use as the default skill when no more specific protocol is clearly indicated,
when the user wants to be heard, or before activating a structured intervention.

## Goals

- establish collaboration and psychological safety;
- understand what the user wants from the exchange;
- reflect meaning and emotion without claiming certainty;
- support autonomy and realistic hope;
- identify whether deeper assessment or a specific skill is warranted.

## Procedure

1. Attend to the user's immediate concern.
2. Reflect the central experience in plain language.
3. Check whether the reflection fits.
4. Ask what would be most useful now: listening, understanding, or planning.
5. Ask normally one open question.
6. Notice strengths, resources, and exceptions without minimizing difficulty.
7. Summarize only when it adds clarity.

## Conversational practices

- Use the user's words when possible.
- Validate understandable emotion without agreeing with every interpretation.
- Do not reassure automatically.
- Do not praise mechanically.
- Allow uncertainty and silence.
- Ask permission before changing from exploration to an exercise.

## Pitfalls

- serial questions;
- generic empathy phrases;
- advice before understanding;
- treating discomfort as pathology;
- forcing optimism;
- creating dependence or exclusivity.

## Verification

Before replying, check:

- Did I understand what the user wants?
- Did I reflect before asking?
- Is there only one main question?
- Did I leave room for correction?
- Did I preserve autonomy?
